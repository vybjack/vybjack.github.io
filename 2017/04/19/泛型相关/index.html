泛型相关

摘要： 认识泛型相关内容

为什么使用泛型编程?

在集合中，存储数据时如果没有指定数据类型，可以存储任意类型的数据，但是在获取的时候，如果不做类型检查就没办法对数据进行处理。所有使用泛型：

  • 获取数据时避免类型检查，提高程序的效率
  • 提高程序的健壮性和安全性
  • 在使用时才指定具体的数据类型，提高灵活性

定义泛型

Person 类引入了一个类型参数 T (Java5 引入的)，类型参数就是用 <> 放在类名后面来表示的，同时可以定义多个类型参数，格式为 < T,K,V >。类型参数可以用来定义方法的返回类型、参数类型、定义域或者局部变量。如：

泛型的类型通配符

< T extends Person> 表示 T 类型是 Person 类型或其子类

如果需要给 T 限定多个类型，则需要使用符号 “&” 如： < T extends Runnable & Serializable>

注意：不管后面的参数类型是接口还是类，都是用 extends 关键字。如果存在类 Class 类型，则至多有一个(Java 的单继承原则)，而且必须放在第一个。

限定类型关键字

  • extends 用来限定子类型 如 Person< T extends Male>
  • super 用来限定超类型 如 Person< T super Animal>
    注意： super 不能用于限定多个类型参数。 如：< T super File & Runnable> //错误

无限定通配符

无限定通配符去除了超类型和子类型的规则，仅仅用一个 “?” 表示，且只能用户类型参数中。如 Person<?>

泛型擦除带来的问题

Person 类泛型擦除后，方法变为 public void setT(Object t){…}
而Male 类中方法还是 public void setT(Dog d){…}
所以 Male 从 Person 中继承了 Public void setT(Object t){…} 而自身有存在 public void setT(Dog d){…} 方法，那么，问题来了

这里 person 引用指向了 Male 实例，根据多态性，setT() 方法应该调用 Male 对象中的方法 setT(Dog d),可是这个擦除让 Person 中的 setT(Object t) 被调用，这就导致了擦除与多态的冲突。此时，虚拟机会在 Male 类中生成一个桥方法(bridge mehtod)。

有了这个桥方法，所以就先调用 setT(Object t) –> setT(Dog d)。

泛型的约束和局限性

  • 不能用基本类型实例化类型参数，如 List// 错误语法

  • 不能实例化类型参数，需要创建 T 对象，必须使用 Java 反射来创建对象，T 的 Class 对象也是不被支持的

  • 不能声明参数化的数组

  • 不能进行类型查询，即不能通过 instanceof 来判断是否属于哪一类型
  • 不能抛出和捕获泛型类实例
